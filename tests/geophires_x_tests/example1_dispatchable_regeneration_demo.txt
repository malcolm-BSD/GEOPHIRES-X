# Example 1 Dispatchable Regeneration Demo
# Based on tests/examples/example1.txt, but configured to show unmet demand
# and shut-in recovery in the dispatchable workflow.
#
# This input demonstrates:
# - dispatchable operating mode
# - cylindrical dispatch adapter support
# - unmet demand when requested load exceeds available geothermal output
# - shut-in recovery during a zero-demand window
# - dispatch-specific summary output
# - improved text output
# - HTML output with optional dispatch graphs
# - hourly dispatch profile CSV export
#
# Recovery is easiest to see in the dispatch profile CSV / HTML graphs:
# - the first 30-day high-demand pulse depletes the reservoir
# - the next 60 days are shut in
# - the second 30-day pulse starts from a hotter recovered state
